# 40-myself 汇报提纲

## 一、从 10-fiddler → 40-myself：重构亮点

### 1. 调度策略解耦 (Strategy Pattern)
```
10-fiddler:   mixtral_forward() 内联 200+ 行 if/else 硬编码调度逻辑
40-myself:    ExpertSchedulingStrategy 基类 + GPUOnlyStrategy / HybridCPUGPUStrategy / PrefetchHybridStrategy
```

- 核心 MoE 循环不再关心"怎么调度"，只关心"调哪个策略"
- 新增策略只需继承基类，零侵入
- `PrefetchHybridStrategy` 返回 4-tuple `(cpu_experts, gpu_experts, prefetch_experts, expert_assignments)`，在原有 CPU/GPU 决策基础上新增预取列表

### 2. CPU/GPU 并行执行 + 预取线程
```
10-fiddler:   串行 —— 先跑完 GPU 专家，再跑 CPU 专家 → 延迟 = T_gpu + T_cpu
40-myself:    3 线程并行 —— ThreadPoolExecutor 同时执行 GPU+CPU+Prefetch → 延迟 = max(T_gpu, T_cpu)，预取在气泡期间完成
```

- 对 CPU 卸载场景收益明显（CPU 耗时被重叠）
- 预取线程利用 I/O 气泡提前加载下一层专家权重到 GPU placeholder

### 3. 代码复用与可测试性
- 工具函数独立：`_build_expert_mask`, `_collect_active_experts`, `_build_latency_lookup`, `_lookup_latency` 等
- 策略类可单独测试，无需加载模型

### 4. 占位符管理器 (ExpertPlaceholderManager)
- 12 个 GPU placeholder + LRU 淘汰策略
- 线程安全：所有公共方法通过 `threading.Lock` 保护，支持预取线程与执行线程并发
- 双向映射：`(layer_id, expert_id) ↔ placeholder` + 按层释放

### 5. Benchmark 驱动的延迟模型
```
20-PDScope:   硬编码 e=1.39, tg=0.95 + 文件级 CPU 延迟表 microdeepseek.txt
40-myself:    benchmark JSON 查找表 {token_count → latency_ms}，运行时自动加载
```
- `t_io` = `expert_weight_copy.avg_ms` ≈ 1.67ms（PCIe 传输，固定值）
- `t_c(tokens)` = `expert_cpu[]` 查表（1→0.14ms, 128→11.58ms，差距 80 倍）
- `t_g(tokens)` = `expert_gpu[]` 查表（1→0.093ms, 128→0.118ms，基本恒定）

## 二、40-myself vs 20-PDScope：已对齐

原计划 3 个关键能力，**已全部完成**：

| # | 差距 | 状态 | 实现内容 |
|---|------|------|----------|
| 1 | **无占位符管理器** | ✅ 已完成 | `ExpertPlaceholderManager`：12 placeholder + LRU 淘汰 + 线程安全锁 + 按层释放 |
| 2 | **无专家预取** | ✅ 已完成 | `GatePredictor` 预测下一层专家 + `_prefetch_next_layer_experts` 并行预取 |
| 3 | **无 PDScope 调度算法** | ✅ 已完成 | `PrefetchHybridStrategy`：Prefill 三步调度法 + Decode ABC 策略 |

### PrefetchHybridStrategy 调度算法详解

**Prefill 阶段（三步调度法）：**
1. **全局排位**：当前层 L + 预测下层 L+1 的非 GPU 常驻专家混合排序，按 token 数升序，通过 `T_all^G < T_all^C` 找全局队列 `L_global`
2. **局部重排**：从 `L_global` 筛选当前层专家，找最小边界使 `T_G < T_C`，得到按需加载集合 `L_on`
3. **置信度感知预取**：计算 I/O 气泡 `T_gap`，通过期望效用 `ξ = R_hit·收益 - (1-R_hit)·损失`，仅 `ξ > 0` 时预取

**Decode 阶段（ABC 策略）：**
- 计算最优 GPU 专家数 `n_g^ρ = argmin max(n_g·t_g, (k-n_g)·t_c)`
- Mode A：当前层 < `n_g^ρ` → 暂停预取保当前层
- Mode B：当前层 > `n_g^ρ`，下层 < `n_g^ρ` → 释放资源预取下层
- Mode C：两层都 > `n_g^ρ` → 只卸载不预取
- 两层都 < `n_g^ρ` → 降级回 Prefill 三步逻辑

### 架构演进
```
10-fiddler:  [硬编码调度]                                     ← 基线
40-myself:   [策略模式调度] + [并行执行]                        ← 骨架
现在:        + [占位符管理] + [预取] + [AdaptSched算法]         ← 对齐 20-PDScope
```

> 关键决策：40-myself 先做架构重构再对齐算法，而不是在混乱代码里再加逻辑。
