# MoE 框架重构计划：闭环自动优化系统

## 目标

重构 MoE GPU 卸载基准测试框架，实现：
1. 统一日志系统，自动检测运行指标并发现调优空间
2. Config 驱动参数传递，microbench 自动生成系统配置
3. 预取策略可开关，通过参数控制

## 现状分析

### 当前问题
- **参数硬编码**: `e`(传输开销)和`tg`(GPU计算开销)硬编码在每个模型文件中，各模型值不同
- **无统一配置**: 无 config 文件机制，cache/batch_size 通过 if-elif 链设置
- **日志散乱**: print 输出遍布代码(含调试 print "111"/"222"等)，日志写到 `./log/linshi.txt`、`./log/expert_stats.txt`、`test.txt` 等多个位置
- **预取策略**: prefetch 代码存在但大部分被注释/禁用，无参数控制开关
- **模型代码高度重复**: deepseek.py/qwen.py/moon.py/mixtral.py 结构几乎相同，核心逻辑重复4次
- **模型选择硬编码**: latency.py 中模型选择逻辑被注释，硬编码为 DeepSeek
- **microbench 输出分散**: 输出到多个文件(micro*_New.txt, hot/*.txt, ioreal_*.png)，未生成结构化配置

---

## 重构方案

### 用户决策
- **模型文件**: 不提取基类，保持四个独立文件
- **Config 格式**: JSON
- **Config 缺失**: 自动运行 microbench 生成
- **日志清理**: 注释掉现有 print + 添加 logger
- **预取**: 启用现有 next-layer gate 预测逻辑 + 添加开关

### 1. 新增 `config.py` — 统一配置管理

```python
# config.py
@dataclass
class ModelConfig:
    model_name: str          # "deepseek", "qwen", "moon", "mixtral"
    model_path: str          # 模型路径
    n_routed_experts: int    # 路由专家数
    n_shared_experts: int    # 共享专家数
    n_layer: int             # 层数
    hidden_size: int         # 隐藏维度
    expert_param_names: list # 权重名 ["gate_proj", "up_proj", "down_proj"] 或 ["w1", "w2", "w3"]
    # microbench 生成的性能参数
    e: float                 # 专家传输开销 (ms)
    tg: float                # 专家GPU计算开销 (ms)
    cpu_time_table: list     # CPU计算时间查找表
    # 运行配置
    batch_size: int
    cache: int
    cpu_offload: int
    beam_width: int
    prefetch_enabled: bool   # 预取策略开关 (默认 False)
    num_placeholders: int    # 占位专家数量 (默认6)
    hot_expert_file: str     # 热点专家文件路径
    cpu_threads: int         # CPU线程数 (默认16)
    # 结果文件路径
    cpu_time_table_file: str # CPU时间查找表文件路径

@dataclass
class SystemConfig:
    gpu_name: str            # GPU型号
    gpu_memory_gb: float     # GPU显存(GB)
    transfer_time_ms: float  # 专家传输时间(ms) → 即 e
    gpu_compute_time_ms: float # GPU专家计算时间(ms) → 即 tg
    cpu_time_table: list     # CPU计算时间表
    attention_time_table: list # 注意力计算时间表
```

功能:
- `load_config(path)`: 从 JSON 文件加载配置
- `save_config(path)`: 保存配置到 JSON 文件
- `from_microbench_results(model_name, micro_dir)`: 从 microbench 结果自动生成配置
- `get_default_config(model_name)`: 返回模型默认配置（硬编码值作为fallback）
- Config 中包含 `e`、`tg`、`cpu_time_table`，microbench 运行后自动更新这些值

### 2. 新增 `logger.py` — 统一日志系统

```python
# logger.py
class RunMetrics:
    """运行时指标收集器"""
    expert_hit_count: int
    expert_total_count: int
    layer_gpu_times: dict     # 每层GPU时间
    layer_cpu_times: dict     # 每层CPU时间
    layer_parallel_times: dict # 每层并行时间
    token_decode_times: list   # 每token decode时间
    expert_stats: list         # 每个专家的统计
    perf_stats: dict           # 各阶段性能统计
    prefill_time: float
    decode_time: float
    
class BenchmarkLogger:
    def __init__(self, config: ModelConfig, log_dir="./logs"):
        self.config = config
        self.metrics = RunMetrics()
        self._log_file = None
        
    def log_layer_stats(self, layer_id, gpu_time, cpu_time, parallel_time, ...):
        """记录每层统计"""
        
    def log_expert_stats(self, layer_id, expert_id, device, token_count, time_ms):
        """记录专家统计"""
        
    def log_token_decode(self, token_idx, time_ms):
        """记录每个token的decode时间"""
        
    def finalize(self) -> BenchmarkReport:
        """运行结束，生成报告"""

class BenchmarkReport:
    """运行报告 + 自动分析"""
    hit_rate: float           # 专家命中率
    parallel_degree: float    # 平均并行度
    throughput: float         # token/s
    avg_decode_time: float    # 平均decode时间
    layer_time_distribution: dict  # 各层时间分布
    expert_classification: dict    # GPU/CPU/ondemand/prefetch 各占比例
    
    def analyze(self) -> list[str]:
        """自动检测调优空间，返回建议列表"""
        # 命中率 < 30% → 建议增加GPU常驻专家数
        # 并行度 < 1.3 → 建议调整e/tg或减少ondemand
        # CPU专家占比 > 50% → 建议增加热点专家
        # 层时间方差大 → 标记瓶颈层
        
    def save(self, path):  # 保存 JSON + TXT
    def print_summary(self):  # 打印摘要
```

报告输出到 `./logs/run_{timestamp}.json` 和 `./logs/run_{timestamp}.txt`

### 3. 新增 `scheduler.py` — 专家调度策略模块

从各模型 `mixtral_forward()` 中提取 ondemand 决策算法，集中管理：

```python
# scheduler.py
class ExpertScheduler:
    """专家调度策略"""
    
    def __init__(self, config: ModelConfig):
        self.config = config
        self.e = config.e
        self.tg = config.tg
        self.cpu_time_table = config.cpu_time_table
        
    def decide_ondemand(self, sorted_experts, is_decode):
        """根据e/tg和CPU时间表决定哪些专家做ondemand
        核心算法来自 deepseek.py 1166-1230行
        """
        n = len(sorted_experts)
        ondemand_experts = []
        TA = sum(self.cpu_time_table[min(tokens, len(self.cpu_time_table)-1)] 
                 for _, tokens in sorted_experts)
        TC = TA
        for i in range(n - 1):
            expert_id, token_count = sorted_experts[i]
            TG = (1 + i) * self.e + self.tg
            TC -= self.cpu_time_table[min(token_count, len(self.cpu_time_table)-1)]
            if is_decode:
                if TG < TC:
                    if token_count > 1:
                        ondemand_experts.append(expert_id)
                    if i == n - 2 and TC - TG > self.e:
                        ondemand_experts.append(sorted_experts[i+1][0])
                else:
                    break
            else:
                if TG < TC + self.cpu_time_table[min(token_count, len(self.cpu_time_table)-1)]:
                    ondemand_experts.append(expert_id)
                else:
                    break
        return ondemand_experts
    
    def should_prefetch(self, layer_id, expert_id):
        """是否预取该专家"""
        return self.config.prefetch_enabled
```

各模型文件中替换硬编码的 `e`/`tg` 和 ondemand 决策逻辑为 `self.scheduler.decide_ondemand()` 调用。

### 4. 各模型文件修改 — 保持独立，注入 config + logger + scheduler

四个模型文件(deepseek/qwen/moon/mixtral)保持独立，但统一：
- `__init__` 接收 `ModelConfig` 而非 `args`
- 内部读取 `self.config.e` / `self.config.tg` 而非硬编码
- 内部读取 `self.config.cpu_time_table` 而非手动打开 `micro*.txt`
- 使用 `self.logger` 替代 print 和手写日志文件
- 使用 `self.scheduler` 替代内联的 ondemand 决策算法
- 预取逻辑根据 `self.config.prefetch_enabled` 控制开关
- 注释掉调试 print，保留关键路径的 logger.info/debug

### 5. 重构 `microbench.py` — 自动生成 config

```python
# microbench.py (重构后)
def run_microbench(model_path: str, output_dir: str = "./configs") -> SystemConfig:
    """运行微基准测试，生成系统配置"""
    
    # 1. 测量专家传输时间 → e (avg_copy_time * 1000 ms)
    # 2. 测量GPU计算时间 → tg (avg_gpu_time * 1000 ms)
    # 3. 测量CPU计算时间表 → cpu_time_table (平滑后的值)
    # 4. 测量注意力时间 → attention_time_table
    # 5. GPU信息 → gpu_name, gpu_memory_gb
    # 6. 生成热点专家列表 → hot/{model}.txt
    
    config = SystemConfig(
        transfer_time_ms=e_ms,
        gpu_compute_time_ms=tg_ms,
        cpu_time_table=cpu_times_smoothed,
        gpu_name=torch.cuda.get_device_name(0),
        gpu_memory_gb=total_mem / 1024**3,
    )
    
    # 保存为 JSON config
    config_path = f"{output_dir}/system_config_{model_name}.json"
    save_config(config, config_path)
    
    # 同时保留原有输出格式（向后兼容）
    # micro{model}_New.txt, ioreal_{model}.png
    
    return config
```

运行 `python microbench.py --model /path/to/model` 后：
- 生成 `configs/system_config_{model}.json` 包含 e, tg, cpu_time_table 等参数
- 保留原有 `micro*_New.txt` 和 `ioreal_*.png` 输出（向后兼容）
- 生成 `hot/{model}.txt` 热点专家列表

### 6. 重构 `latency.py` — Config 驱动 + 闭环

```python
# latency.py (重构后)
def main():
    args = parse_args()
    
    # 1. 加载或自动生成config
    config = load_or_generate_config(args)
    # load_or_generate_config:
    #   - 优先从 --config 参数加载
    #   - 否则查找 configs/system_config_{model_name}.json
    #   - 找不到则自动运行 microbench 生成
    
    # 2. 自动选择模型类
    model_cls = get_model_class(config.model_name)
    model = model_cls(config)
    
    # 3. 运行benchmark
    report = run_benchmark(model, config)
    
    # 4. 输出报告
    report.print_summary()
    report.save()
    
    # 5. 自动检测调优空间
    suggestions = report.analyze()
    if suggestions:
        print("\n=== 调优建议 ===")
        for s in suggestions:
            print(f"  - {s}")
```

命令行参数：
- `--model`: 模型路径（现有）
- `--batch_size`: 批大小（现有）
- `--cpu-offload`: CPU卸载开关（现有）
- `--cache`: 预取缓存大小（现有）
- `--config`: 指定config文件路径（新增）
- `--prefetch`: 预取策略 enabled/disabled（新增，默认disabled）
- `--log-dir`: 日志目录（新增，默认./logs）

### 7. 预取策略参数化

在 config 和命令行中新增 `--prefetch` 参数：
- `--prefetch disabled`: 禁用预取（当前默认行为，保持兼容）
- `--prefetch enabled`: 启用next-layer gate预测预取

在各模型的 `mixtral_forward()` 中，启用现有的预取逻辑代码（取消注释），通过 `self.config.prefetch_enabled` 控制是否执行。

## 关键设计决策

1. **Config 格式选择 JSON**: Python内置支持，无需额外依赖
2. **不改动 micro*.txt 格式**: 向后兼容，microbench 同时输出 txt 和 json
3. **Logger 输出双格式**: JSON(机器解析) + TXT(人类阅读)
4. **不提取模型基类**: 保持四个独立文件，减少重构风险，各文件自行注入config/logger/scheduler
5. **预取策略默认关闭**: 保持当前行为，通过 `--prefetch enabled` 启用
6. **e/tg 来源优先级**: config JSON > microbench 自动生成 > 硬编码默认值
7. **注释而非删除print**: 保留调试信息，但不影响性能
8. **Config缺失自动运行microbench**: 实现闭环，首次运行自动生成配置
