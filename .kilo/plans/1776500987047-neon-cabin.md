# Plan: 复现微基准测试（Point 1）

## 目标
独立编写新的微基准测试代码，测量 MoE 专家的三个关键性能参数：
1. **`e`** — 专家权重从 CPU 搬运到 GPU 的传输时间（ms）
2. **`tg`** — 专家在 GPU 上的计算时间（ms）
3. **`cpu_time_table`** — 不同 token 数下专家在 CPU 的计算时间表（ms）

覆盖两个模型：DeepSeek-v2-lite-chat 和 Qwen3-30B-A3B。

---

## 新代码文件

创建 `opensource/microbench_v2.py` — 完全独立的新实现，不依赖现有 `microbench.py`。

## 实现步骤

### Step 1: 框架搭建
- 参数解析（`--model` 指定模型路径，`--output-dir` 指定输出目录）
- 从模型路径自动推断模型名称（deepseek/qwen/moon/mixtral）
- GPU 信息检测（`torch.cuda.get_device_name`, 显存大小）
- 模型加载：`transformers.AutoModelForCausalLM.from_pretrained()`

### Step 2: 提取专家模块
- 从模型的第 1 层（layer index 1）获取专家列表
- DeepSeek/Qwen: `model.model.layers[1].mlp.experts`，权重名为 `gate_proj`, `up_proj`, `down_proj`
- 创建 GPU 上的占位专家（`copy.deepcopy(experts[0]).to("cuda:0")`），用于接收 CPU 拷贝

### Step 3: 测量 `e`（CPU→GPU 传输时间）
- 将专家[2]移到 CPU
- 对其三个权重矩阵调用 `.pin_memory()`（加速传输）
- 使用 `torch.cuda.synchronize()` + `time.time()` 精确计时
- 将三个权重从 CPU 拷贝到 GPU 占位专家（`dst.copy_(src)`）
- 重复 64 次（token_counts = 1..64），取平均值作为 `e`
- **本质**：测量的是 PCIe 带宽相关的纯传输时间，与 token 数无关

### Step 4: 测量 `tg`（GPU 计算时间）
- 将专家[1]移到 GPU
- 对每个 token 数（1..64）：
  - 生成随机输入：`torch.randn((token_count, hidden_size), dtype=dtype, device="cuda")`
  - Warmup 运行 1 次
  - 计时运行 1 次（`torch.cuda.synchronize()` 包裹）
  - 记录时间
- 取 64 次测量的平均值作为 `tg`
- **本质**：GPU 上专家 MLP 前向计算时间

### Step 5: 测量 `cpu_time_table`（CPU 计算时间表）
- 将专家[1]移到 CPU
- 对每个 token 数（1..64）：
  - 生成随机输入：`torch.randn((token_count, hidden_size), dtype=dtype, device="cpu")`
  - Warmup 运行 1 次
  - 计时运行 **20 次**，取平均值
- 对结果做滑动窗口平滑（window_size=7）
- 前部缺失用线性插值补齐
- **本质**：CPU 上专家 MLP 前向计算时间，与 token 数强相关（线性增长）

### Step 6: 输出结果
1. **`micro{model_name}_v2.txt`** — CSV 格式（Token Count, e, tg, cpu_time, attn_time）
2. **`configs/system_config_{model_name}_v2.json`** — JSON 系统配置
3. **`ioreal_{model_name}_v2.png`** — 性能折线图（matplotlib）
4. 终端打印：e、tg、GPU 型号等摘要信息

### Step 7: 验证
- 运行 DeepSeek 模型的微基准测试
- 运行 Qwen 模型的微基准测试
- 将结果与现有 `microdeepseek.txt` / `microqwen.txt` 对比，验证数量级一致

## 关键技术细节

| 项目 | DeepSeek | Qwen |
|------|----------|------|
| 模型路径 | `/mnt/g/Models/DeepSeek-v2-lite-chat` | `/mnt/g/Models/Qwen3-30B-A3B` |
| 路由专家数 | 64 | 128 |
| 共享专家数 | 2 | 0 |
| 层数 | 26 | 48 |
| hidden_size | 2048 | 2048 |
| 权重名 | gate_proj, up_proj, down_proj | gate_proj, up_proj, up_proj |
| dtype | bfloat16 | bfloat16 |
| 预期 e | ~1.1 ms | ~0.55 ms |
| 预期 tg | ~0.2 ms | ~0.85 ms |

### 计时注意事项
- GPU 计时必须用 `torch.cuda.synchronize()` 包裹，否则异步执行导致计时不准
- CPU 传输用 `pin_memory()` 加速，这是真实推理场景下的优化
- CPU 计时也要 `torch.cuda.synchronize()`（确保没有 GPU 操作干扰）

### 平滑算法
```python
cpu_times_smoothed = np.convolve(cpu_times, np.ones(7)/7, mode="valid")
# 前 6 个点用线性插值补齐
for i in range(6):
    cpu_times_full[i] = cpu_times_smoothed[0] * (i + 1) / 7
```
