## 阶段 1: Multi-stream 并发传输（优先级最高，独立性强）

**目标**: 将 `gate_proj / up_proj / down_proj` 的串行 `copy_()` 改为 3 个 CUDA Stream 并行传输

### 1.1 在 `__init__` 中创建传输 streams
```python
self._transfer_streams = [torch.cuda.Stream() for _ in range(3)]
```

### 1.2 修改 `_async_ondemand()` (行 402-438)
将串行拷贝：
```python
for name in ["gate_proj", "up_proj", "down_proj"]:
    dst.copy_(src)
```
改为多流并行：
```python
for stream, name in zip(self._transfer_streams, ["gate_proj", "up_proj", "down_proj"]):
    with torch.cuda.stream(stream):
        dst.copy_(src, non_blocking=True)
for s in self._transfer_streams:
    torch.cuda.synchronize(s)
```

### 1.3 修改 `_async_load_expert()` (行 440-503)
同样替换拷贝逻辑，使用多流 + prefetch_stream.wait_stream() 确保同步。

### 1.4 预期收益
传输时间 ≈ max(单矩阵拷贝) ≈ 1/3 串行时间。

## 阶段 3: Prefill 三步调度

**目标**: 实现 "全局排序 → 局部重排 → 置信度预取(ξ)" 的 prefill 专用调度

### 3.1 全局排序 (Step 1)
合并当前层 + 下层预测专家，按 token 数降序全局排序。

### 3.2 局部重排 (Step 2)
确保当前层专家不被饿死 — 当前层专家优先调度。

### 3.3 置信度预取 (Step 3)
计算期望效用 ξ = R_hit · f · e - (1-R_hit) · max(0, i-f+1) · e
利用 I/O bubble 时空闲带宽预取下层专家。
